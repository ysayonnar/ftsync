Сборка и запуск:
1. make build
2. ./build/daemon - запуск демона
3. ./build/cp - запуск клиентской программы

после запуска клиентской программы необходимо ввести хост и порт, но если нажать "enter" сразу, то программа подключиться к стандартному хосту и порту демона

Сейчас реализованы 3 команды:
1. ping - для проверки соединения
2. ls - для просмотра содержимого текущего каталога демона
3. cd <dirname> - для навигации по дереву каталогов демона


Тема:

Многопоточная переносимая программа управления обменом файлами. 
Представляет собой реализацию оригинального прикладного протокола над TCP (прикладной протокол разрабатывается студентом). 
Состоит из консольной управляющей программы (CP) и демона. CP инициирует обмен файлами между двумя хостами, на каждом из которых установлен демон. 
CP реализует ряд команд, позволяющих перемещаться по дереву файловой си- стемы демона, а также инициировать пересылку файлов и каталогов от одного демона к другому.
Стороны обмена используют средства контроля целостности и продолжение передачи с контрольной точки в случае обрыва передачи/приема.
В процессе установле- ния соединения программы с демоном использовать протокол аутентификации на осно- ве UUID и RSA (uuid/uuid.h, openssl/ssl.h).
